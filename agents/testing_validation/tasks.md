# Testing & Validation Tasks

## Draft Test Plan

Input: requirements, acceptance criteria, and the implementation.

Output: test plan mapping each criterion to unit, integration, and regression
tests, plus edge cases.

## Review Test Suite

Input: existing tests and the code under test.

Output: coverage and gap analysis, missing edge cases, and non-deterministic
test risks.

## Validate Model Or Backtest

Input: model/backtest outputs and the claims made about them.

Output: validation review covering splits, leakage, look-ahead, sample size, and
statistical significance.

## Write Regression Tests

Input: a fixed bug or a behavior that must not regress.

Output: deterministic regression tests that fail on the old behavior.
