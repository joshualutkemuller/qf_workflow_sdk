# Maintenance & Monitoring Tasks

## Design Monitoring Plan

Input: live system, baselines, and known failure modes.

Output: monitoring and health-check plan with metrics, thresholds, and alerts.

## Triage Drift Or Decay

Input: observed performance or data change and recent history.

Output: triage separating drift, regime change, and noise, with root-cause
analysis and a recommended action.

## Write Postmortem

Input: an incident, its timeline, and its impact.

Output: postmortem with root cause, contributing factors, and dated, owned
action items.

## Refresh Stale Documentation

Input: live system and its current runbooks, model cards, and dataset cards.

Output: updated documentation that matches the running system and open gaps.
