# Coding & Implementation Tasks

## Implement From Design

Input: approved design, target codebase, and conventions.

Output: implementation faithful to the design with reproducibility notes and a
list of what still needs tests.

## Promote Notebook To Module

Input: notebook cells and the logic to reuse.

Output: importable, reproducible module with pinned inputs and no hidden state.

## Self-Review A Diff

Input: a proposed change or diff.

Output: review covering design fidelity, conventions, leakage, look-ahead,
reproducibility, and secret exposure.

## Refactor For Reproducibility

Input: working but fragile code with hidden state or unpinned inputs.

Output: refactored code with explicit inputs, seeds, and deterministic outputs.
