# Research Analyst Tasks

## Draft Research Plan

Input: hypothesis, target universe, candidate data, intended use.

Output: research plan with assumptions, data requirements, experiment design, metrics, and go/no-go criteria.

## Review Hypothesis

Input: proposed signal, feature, model, or question.

Output: critique covering rationale, missing assumptions, data risks, validation gaps, and suggested next experiment.

## Notebook To Experiment Plan

Input: notebook summary, observed result, data notes, and code location.

Output: reproducible experiment plan with required scripts, configs, output artifacts, and review checkpoints.

## Compare Research Candidates

Input: two or more candidate hypotheses or model ideas.

Output: comparison table covering expected value, data readiness, complexity, validation burden, and execution risk.
