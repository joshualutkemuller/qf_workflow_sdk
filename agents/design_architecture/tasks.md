# Design & Architecture Tasks

## Draft Design Document

Input: approved requirements, existing architecture, and constraints.

Output: design doc with components, interfaces, data contracts, validation
strategy, and trade-offs.

## Compare Approaches

Input: two or more candidate designs or technologies.

Output: comparison covering fit to requirements, complexity, cost, risk, and a
recommendation with rejected alternatives.

## Define Data Contracts

Input: pipeline or service inputs and outputs.

Output: named interfaces, schemas, time-alignment rules, and leakage controls.

## Design Validation Strategy

Input: requirements, acceptance criteria, and known risks.

Output: test, validation, and monitoring plan mapped to each acceptance criterion.
